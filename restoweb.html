RestoWeb


Ce projet est un site web de critique de restaurant afin de récupérer les avis des consommateurs il a été réalisé afin de montrer comment fonctionne un site web de type MVC, MVC étant l'abréviation de Modèle, Vue et Controleur
Le modèle représente l'endroit ou toutes les données sont.
La vue représente la partie visible de l'application donc le design.
Le controleur représente l'endroit ou les données sont récupérées et traité

Le controleur permet donc de partager des données à afficher par la vue et les données traités par le controleur sont contenus par le modèle.

Design du système de filtre


Pourquoi utiliser une architecture de type MVC ?


Le principal avantage d'une structure MVC est son ergonomie en effet séparer les différentes données permet de mieux organiser le code et donc de savoir précisément à quel endroit tel fonction se réalise

Un autre avantage de cette séparation est que l'on peut réutiliser ou faire un évoluer une partie du code par exemple une vue peut être semblable à une autre mais afficher des données différentes et peut être modifié indépendamment du reste, cela permet aussi de mieux réaliser les tests unitaires.

Il existe aussi un avantage si la réalisation du site web se fait en équipe, car cela permet de répartir les tâches entre différents développeurs l'un réalisant la partie controleur, l'autre la vue, et un autre le modèle

En résumé réaliser un site Web sous l'architecture MVC permet de séparer les différentes parties du code afin de le rendre plus ergonomique et permet de mieux contrôler chaquer partie indépendamment du reste, il est donc très utile pour des très grands projets.
